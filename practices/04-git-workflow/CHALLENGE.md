## Step 5: /rewind 연습

### 5-1. 의도적인 변경

## Tips

- 변경 전 항상 백업 브랜치를 만드는 습관을 들이세요
- Claude에게 "커밋 메시지를 작성해줘"라고 하면 변경 내용을 분석하여 적절한 메시지를 작성합니다
- `/rewind`는 Claude의 모든 변경 사항(파일 수정, Git 작업)을 되돌릴 수 있습니다
- 위험한 Git 작업(force push, reset --hard 등)은 Claude가 경고해줍니다
