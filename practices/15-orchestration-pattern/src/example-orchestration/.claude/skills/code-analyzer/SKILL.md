# Code Analyzer Skill

## Purpose

TypeScript/JavaScript 코드의 품질을 분석하고 점수를 산정합니다.
이 Skill은 분석 기준과 점수 산정 규칙을 정의합니다.

## Analysis Criteria

### 1. Complexity (복잡도) — 30점

| 항목 | 기준 | 점수 |
|------|------|------|
| 함수 길이 | 20줄 이하 | 10점 |
| | 21-50줄 | 5점 |
| | 50줄 초과 | 0점 |
| 중첩 깊이 | 3단계 이하 | 10점 |
| | 4-5단계 | 5점 |
| | 6단계 이상 | 0점 |
| 매개변수 수 | 3개 이하 | 10점 |
| | 4-5개 | 5점 |
| | 6개 이상 | 0점 |

### 2. Maintainability (유지보수성) — 30점

| 항목 | 기준 | 점수 |
|------|------|------|
| 타입 안전성 | `any` 미사용 | 10점 |
| | `any` 1-2회 | 5점 |
| | `any` 3회 이상 | 0점 |
| 에러 처리 | 적절한 try-catch | 10점 |
| | 일부 누락 | 5점 |
| | 에러 처리 없음 | 0점 |
| 네이밍 | 의미있는 이름 | 10점 |
| | 일부 불명확 | 5점 |
| | 단일 문자 변수 다수 | 0점 |

### 3. Best Practices (모범 사례) — 40점

| 항목 | 기준 | 점수 |
|------|------|------|
| 단일 책임 원칙 | 준수 | 10점 |
| | 일부 위반 | 5점 |
| | 다수 위반 | 0점 |
| DRY 원칙 | 중복 없음 | 10점 |
| | 약간의 중복 | 5점 |
| | 상당한 중복 | 0점 |
| 테스트 존재 | 테스트 파일 있음 | 10점 |
| | 일부만 있음 | 5점 |
| | 테스트 없음 | 0점 |
| 문서화 | JSDoc 있음 | 10점 |
| | 일부만 있음 | 5점 |
| | 문서 없음 | 0점 |

## Patterns to Detect

### Red Flags (감점 요소)
- `console.log` 디버깅 코드 방치
- `// TODO` 주석이 3개 이상
- 하드코딩된 매직 넘버
- 사용되지 않는 import
- `@ts-ignore` 사용

### Green Flags (가점 요소)
- 일관된 코딩 스타일
- 의미있는 커밋 메시지
- 인터페이스/타입 분리
- 적절한 추상화 레벨

## Output Format

### 파일별 보고서

```
| 파일 | 복잡도 | 유지보수성 | 모범사례 | 총점 | 등급 |
|------|--------|-----------|---------|------|------|
| file.ts | X/30 | X/30 | X/40 | X/100 | A~F |
```

### 등급 기준

- **A** (90-100): 우수 — 모범적인 코드
- **B** (80-89): 양호 — 약간의 개선 여지
- **C** (70-79): 보통 — 개선 필요
- **D** (60-69): 미흡 — 상당한 개선 필요
- **F** (0-59): 불량 — 즉시 리팩토링 필요

### 전체 요약

```
프로젝트 평균: X/100 (등급)
분석 파일 수: N개
개선 필요 파일: N개
```
